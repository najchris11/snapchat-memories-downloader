# AGENTS.md

## Project map
- Main product is the Kotlin Multiplatform app in `composeApp/` (`commonMain` shared logic/UI, `desktopMain` desktop actuals, `androidMain` Android actuals).
- The end-to-end pipeline starts in `composeApp/src/commonMain/kotlin/com/najdev/snapvault/App.kt`: `HistoryParser` → `DownloadEngine` → optional `MediaProcessor` metadata injection → `Deduplicator` → `vault_index.json`.
- Desktop library browsing is backed by `composeApp/src/desktopMain/kotlin/com/najdev/snapvault/MediaScanner.kt`, which reads `vault_index.json` and exposes `scanMediaFiles()` / `loadThumbnail()`.

## Important boundaries
- Keep shared business logic in `commonMain`; platform-specific filesystem, pickers, and binaries belong in `desktopMain` / `androidMain` via `expect` / `actual` (`PlatformPickers.kt`, `MediaScanner.kt`, `ThemePreferences.kt`).
- `MemoryItem` and `FileMeta` are the persisted data contracts; changing fields affects JSON parsing and `vault_index.json` compatibility.
- UI log strings are tag-based (`[INFO]`, `[SUCCESS]`, `[WARN]`, `[DOWNLOADED]`, `[DELETED DUPES]`, `[METADATA]`) because `DashboardScreen.kt` parses them for styled display.

## Desktop-specific workflow
- `composeApp/src/desktopMain/kotlin/com/najdev/snapvault/Main.kt` launches the Compose desktop app; `MainCli.kt` is a separate CLI runner for quick local pipeline checks.
- `BinaryExtractor.checkCommand()` auto-discovers `exiftool` / `ffmpeg` from PATH, `~/.snapvault/bin`, or bundled zips under `composeApp/src/desktopMain/resources/bin/<platform>/`.
- If you update bundled tools, regenerate them with `scripts/prepare-runtime-macos.sh`, `scripts/prepare-runtime-linux.sh`, or `scripts/prepare-runtime-windows.ps1` rather than editing extracted binaries by hand.

## Legacy tooling
- The Python orchestrator in `legacy/run_all.py` chains the old scripts: `snapchat-downloader.py`, `metadata.py`, `combine_overlays.py`, and `delete-dupes.py`.
- The legacy Electron/Next app lives in `legacy/gui/`; its useful scripts are `npm run electron-dev` and `npm run electron-build`.

## Build and test commands
- Shared Kotlin tests: `./gradlew test`.
- Desktop-only checks: `./gradlew :composeApp:desktopTest`.
- Desktop CLI smoke run: `./gradlew :composeApp:runCli`.
- Legacy pipeline smoke run: `python3 legacy/run_all.py --full` (expects `memories_history.html` in the repo root).
- GUI dev: `cd legacy/gui && npm install && npm run electron-dev`.

## Repo conventions to preserve
- Preserve the parser’s dual support for Snapchat HTML exports and `memories_history.json` (`HistoryParser.parse()` / `parseJson()`).
- Keep filename generation stable: `DownloadEngine.buildFilename()` uses date-derived prefixes like `20231012_153000_<id>.jpg` and resume detection relies on those patterns.
- `Deduplicator` keeps one file per hash, preferring filenames that start with the folder UUID when present; its `dryRun` mode must stay non-destructive.
- `AppBuildConfig.VERSION` / `IS_DEBUG` are generated by Gradle in `composeApp/build.gradle.kts`; don’t hardcode them in UI files.
- Avoid modifying `build/`, generated resources, or runtime artifacts unless you are intentionally regenerating them.
